Right now:

Problem:
	•	You sometimes don’t have final collections yet
	•	You need to operate off modeled revenue early

✅ Add This

grossRevenue: number | null
costAmount: number | null
netRevenue: number

GAP B — No Rep Ownership History
Current:
Uses organizations.owner_user_id
Problem:
	•	Ownership can change mid-month
	•	You lose attribution accuracy

✅ Add This
CommissionRecord {
  ownerRepUserId
  ownerRepSnapshotName
}

Why
	•	Locks who earned it at time of calculation
	•	Prevents disputes

GAP C — No Multi-Rep Split Support

Problem:

	◦	Source deal
	◦	Close deal

GAP D — Missing “Commission Source Type”
Right now:
Everything flows into CommissionRecord
Problem:
	•	You can’t distinguish:
	◦	EMS auto
	◦	Event manual
	◦	Education manual

✅ Add
sourceType: "EMS_AUTO" | "MANUAL_EVENT" | "MANUAL_EDU" | "MANUAL_GOV"

Why
	•	Reporting clarity
	•	Debugging
	•	Future automation

GAP E — No “Period Close” Enforcement
You have:
Lock period
Missing:
	•	Hard prevention of:
	◦	recalculation
	◦	ledger edits
	◦	rule changes affecting past data

✅ Add Rule
Locked period = immutable

Add this constraint:
If period is LOCKED, no ledger edits, rule edits, or recalculations can affect that period. Adjustments must be used instead.

GAP F — Missing “Negative Net Revenue Handling”
Current:
commission = net × rate
Problem:
	•	What if net is negative?

✅ Define behavior
commissionBase = max(netRevenue, 0)

Why
	•	Prevents negative payouts
	•	Keeps logic predictable
