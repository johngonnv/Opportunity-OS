Scan Structure Logic

Step 1 — Internal Master Lookup

Given the current organization:
	•	org name
	•	normalized name
	•	website
	•	domain
	•	Google place data if available

Check master database first for:
	•	exact canonical name
	•	alias match
	•	normalized name similarity
	•	domain match
	•	known corporate family tree relationships

If strong match exists:
	•	return it as the primary suggestion
	•	do not run external enrichment unless needed

Step 2 — External Validation (fallback)

If no strong internal match exists:
	•	search external sources for parent-company clues

Initial external signals can include:
	•	official website domain analysis
	•	Google Places / Place Details data already available
	•	public business/entity sources
	•	structured web results if already supported by backend

Do not auto-apply external results.

Step 3 — LLM Reasoning Layer

If multiple weak or mixed candidates exist:
	•	use LLM reasoning to compare candidate entities and summarize likely relationships

LLM use cases:
	•	compare similar company names
	•	detect likely parent/child relationship language
	•	summarize evidence into a human-readable explanation
	•	recommend confidence level

LLM must NOT be treated as the legal source of truth.
It is an interpretation layer only.

Step 4 — Review Result

Return:
	•	suggested parent
	•	suggested ultimate parent
	•	suggested structure type
	•	confidence score
	•	evidence summary
	•	source breakdown

Step 5 — User Approval

User actions:
	•	Approve
	•	Reject
	•	Save for later
	•	Create draft parent manually

Step 6 — Apply Approved Structure

On approval:
	•	update workspace org hierarchy fields
	•	preserve cycle detection
	•	create activity log
	•	create audit log
	•	optionally write approved mapping back to master graph

Practical cost view

Because you asked about using ChatGPT/OpenAI for this: use it selectively. OpenAI API costs are usage-based, so make it a secondary reasoning step, not the default first lookup. Your cheapest lookup is your own master database; your next cheapest is deterministic matching; the most expensive layer should be LLM synthesis.
