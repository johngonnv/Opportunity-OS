Replit — Phase 3 Admin Review Upgrade Directive
Goal: upgrade the current `/admin/onboarding/[id]/review` screen from a structured approval UI into a true Opportunity OS strategic configuration engine.

The current build is good structurally:
- grouped review sections
- blocking items
- confidence labels
- approve / edit / reject
- sticky action bar

Keep all of that.

Now implement the following upgrades so this phase behaves like true Opportunity OS:
- AI suggests
- admin decides
- system explains impact
- provisioning creates execution-ready workspaces
- empty intelligence gaps are exposed and guided, not left dead

==================================================
1) CORE UPGRADE OBJECTIVE
==================================================

We need this phase to do 4 things at once:

1. Validate AI suggestions
2. Guide admin when AI is weak or empty
3. Show the downstream impact of each decision
4. Seed execution, intelligence, and master-database growth at provisioning time

This phase must feel like:
"Configure how this client will actually win business."

Not:
"Review a few CRM fields."

==================================================
2) KEEP CURRENT STRUCTURE
==================================================

Keep these existing groups:
- Classification
- Business Model
- Market Strategy
- Execution Layer
- Intelligence Layer
- Tagging
- Add-Ons
- Risk / Warnings

Keep:
- blocking list at top
- confidence badges
- approve / edit / reject actions
- sticky bottom CTA
- mobile-first card layout

Do not redesign from scratch.
Extend the current implementation.

==================================================
3) REQUIRED UX UPGRADE — SHOW SUGGESTED VS FINAL
==================================================

Right now cards mostly show only:
- AI: value

This is not enough.

Every review item card must show both:
- Suggested Value
- Final Value

Rules:
- If item is untouched, Final Value should display as `Not finalized`
- If admin approves AI as-is, Final Value becomes the suggested value
- If admin edits, Final Value shows edited value
- If admin rejects, Final Value remains empty until replaced

Required display structure on item cards:

Label
Confidence
Status

AI Suggestion:
<suggested value or empty>

Final Value:
<final reviewed value or "Not finalized">

Actions:
[Approve] [Edit] [Reject]

Also show:
- last reviewed by
- last reviewed at

==================================================
4) REQUIRED UX UPGRADE — HANDLE EMPTY OR WEAK AI SUGGESTIONS CORRECTLY
==================================================

Current problem:
Some critical fields are blank but still show Approve.

This is wrong.

New rule:
If suggested value is null / empty AND final value is null / empty:
- disable Approve
- show `Needs Input`
- force Edit flow

Examples:
- Target Facilities
- Sales Motions
- Competitors
- Pain Points

UI behavior:
- Replace green Approve button with disabled or muted state
- Add helper text:
  `AI could not confidently determine this. Admin input required.`

==================================================
8) REQUIRED BUSINESS LOGIC — CONFIDENCE-BASED DECISION SUPPORT
==================================================

Do not treat all fields equally.

Implement confidence-sensitive behavior:

HIGH confidence:
- easy approve allowed
- show as one-tap action

MEDIUM confidence:
- approve allowed
- encourage review with suggested alternatives

LOW confidence:
- show caution state
- encourage edit before approval

EMPTY suggestion:
- no direct approve
- must edit

This means:
- empty market strategy should require human input
- empty intelligence layer should require human input
- high-confidence classification can remain fast to approve

11) REQUIRED AUDIT / LEARNING UPGRADE
==================================================

For every review item, store and expose:
- suggested value
- final value
- whether admin approved as-is or edited
- rejection reason if rejected
- delta between AI suggestion and final value

Add a simple internal field:
- `decision_type`
Values:
- APPROVED_AS_SUGGESTED
- EDITED_FROM_SUGGESTION
- MANUALLY_CREATED
- REJECTED

This data is important for:
- future AI tuning
- admin auditability
- learning which recommendation categories are weak

12) REQUIRED ITEM CARD RULES
==================================================

Update item cards with these rules:

If suggested value exists:
- show AI Suggestion
- allow Approve / Edit / Reject

If suggested value is empty:
- show `AI Suggestion: None`
- show helper text:
  `Admin input required`
- disable Approve
- keep Edit and Reject

If final value exists:
- show it prominently

If final value differs from suggested value:
- show a subtle badge:
  `Edited from AI`

If status is REJECTED:
- require rejection reason
- show reason on expansion

15) REQUIRED VALIDATION RULES
==================================================

Apply & Provision must stay blocked unless all required fields are truly resolved.

A required item is resolved only if:
- it is APPROVED and has a final value
OR
- it is EDITED and has a final value

A required item is NOT resolved if:
- status = PENDING
- status = REJECTED
- final value is empty

For empty-suggestion required items:
- only Edit can resolve them
