* organization enrichment

## Core Logic

**NAICS = what the organization is**
**PSC = what the government buys**

Both are required.

# 2) CREATE MASTER TABLES

## 2A) NAICS MASTER

Create:

```sql
naics_master
- code text primary key
- title text not null
- description text
- parent_code text
- level integer
- sector_code text
- source_file text
- created_at timestamp
- updated_at timestamp
```

### NAICS ingestion rules

* Only valid 6-digit codes from `6-digit_2022_Codes.xlsx` can be assigned to organizations
* Use `2022_NAICS_Structure.xlsx` to derive:

  * parent_code
  * level
  * sector_code
* Use `2022_NAICS_Descriptions.xlsx` to populate description
* Normalize imported values:

  * trim whitespace
  * strip trailing marker characters like `T`
  * preserve leading zeros if any string fields need them
* Reject duplicates
* Do not generate NAICS outside uploaded files

# 6) NAICS + PSC INFERENCE ENGINE

When an organization is created, updated, scanned, or enriched, run classification for both NAICS and PSC.

## Inputs

* organization.name
* organization.description
* organization.website
* organization.notes
* google_place_category if available
* existing tags or service line labels if available

## REQUIRED PROCESSING ORDER

### NAICS

1. keyword match using `naics_keyword_map`
2. semantic similarity against `naics_master.description`
3. category mapping from Google Places / org type
4. score + rank
5. validate against `naics_master`
6. save top result as primary and additional strong matches as secondary

### PSC

1. keyword match against PSC name, description, includes, excludes, notes
2. semantic similarity against `psc_master.full_description`
3. service/buying pattern inference from org description and notes
4. score + rank
5. validate against `psc_master`
6. save top result as primary and additional strong matches as secondary

## Output shape

Use strict structured output internally like:

```json
{
  "primary_naics": {
    "code": "621910",
    "title": "Ambulance Services",
    "confidence": 0.94,
    "rationale": "Matches ambulance and medical transport language"
  },
  "secondary_naics": [],
  "primary_psc": {
    "code": "V225",
    "title": "Transportation/Travel/Relocation- Travel/Lodging/Recruitment: Travel/Lodging",
    "confidence": 0.71,
    "rationale": "Closest buying pattern match based on travel and transport support language"
  },
  "secondary_psc": []
}
```

## Fallback rule

If AI is unavailable, system must still work using deterministic keyword/rule logic.

## Review rule

If confidence < 0.70, still save suggestion but flag as needs review.

# 7) GOVCON RADAR

Build an initial GovCon radar that uses both NAICS and PSC.

## Create

```sql
govcon_opportunities
- id uuid primary key
- title text not null
- naics_code text
- psc_code text
- agency text
- region text
- prime_or_sub_fit text
- summary text
- source text
- created_at timestamp
- updated_at timestamp
```

Mock data is acceptable for now, but the logic must be real.

## Matching logic

An opportunity should score against:

* workspace target NAICS
* workspace target PSC
* region
* agency target
* prime/sub fit

## Scoring weights

| Factor        | Weight |
| ------------- | -----: |
| PSC match     |     35 |
| NAICS match   |     30 |
| Region match  |     15 |
| Agency match  |     10 |
| Prime/Sub fit |     10 |

## Output

For each matched opportunity, calculate:

* `opportunity_score` (0–100)
* `match_reason`
* `recommended_action`
