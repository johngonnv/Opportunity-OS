Replit Build Spec — Opportunity OS MVP
Project summary

Build a non-PHI CRM web app called Opportunity OS.

The app starts as a business card scanner + CRM + follow-up engine.
It must be designed so it can later expand into:

GovCon opportunity management
Healthcare business development
HIPAA-separated compliant modules later

This MVP must not store PHI.
It is strictly for:

business contacts
companies / organizations
notes without patient-identifiable info
opportunities
tasks
activities
scanned business cards
Product goals
Turn scanned business cards into structured CRM records
Detect duplicates
Create contacts and organizations
Track activities, notes, and tasks
Manage opportunities through pipeline stages
Provide dashboard visibility
Be clean, modern, and easy to extend
Tech stack
Frontend: Next.js 14+ App Router with TypeScript
Backend: Next.js server actions and route handlers
Database: PostgreSQL
ORM: Prisma
Auth: NextAuth or Clerk
UI: Tailwind CSS + shadcn/ui
Validation: Zod
Tables: TanStack Table
State: minimal local state, server-first
File uploads: local abstraction with ability to swap to cloud later
OCR integration: abstracted OCR service interface
Icons: lucide-react
Design requirements
Clean dashboard-style interface
Sidebar navigation
Responsive layout
Neutral professional styling
Avoid flashy colors
Use rounded cards, clear spacing, easy-to-scan tables
Support desktop first, mobile second
Compliance constraint

This MVP must not collect or store:

patient names
DOB
diagnoses
insurance member info
medical record numbers
identifiable transport episode details
any PHI

Add visible warning text in note-entry areas:
“Do not enter patient-identifiable or health information.”
